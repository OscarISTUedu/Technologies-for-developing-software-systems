# Система мониторинга инклинометров на предприятиях нефтегазового комплекса 

## Описание проекта

Данный проект представляет собой серверную часть распределенной системы для сбора, хранения и анализа данных с инклинометров, используемых в геологоразведочной деятельности предприятия "Сибирский меридиан".

Система состоит из трех основных компонентов:
1.  **Инклинометры** - измерительные устройства, с заданным интервалом регистрирующие показания ускорения (для определения угла наклона) и температуры.
2.  **Базовая станция (концентратор)** - микроконтроллер, выступающий в роли промежуточного узла для сбора данных с инклинометров и их передачи на сервер.
3.  **Сервер** - центральный узел системы, обеспечивающий прием, хранение, обработку данных и предоставление к ним удаленного доступа.

Данный репозиторий содержит код серверного сервиса, отвечающего за функционирование третьего компонента.

## Основной функционал (Задачи дипломного проекта)

*   **Прием данных:** Разработка API для приема потоков данных от базовых станций.
*   **Хранение данных:** Проектирование и реализация структуры базы данных для эффективного хранения временных рядов показаний инклинометров.
*   **Удаленный доступ:** Настройка сетевой инфраструктуры для безопасного удаленного подключения к серверу и взаимодействия с системой.
*   **Безопасность:** Реализация механизмов аутентификации и авторизации для разграничения доступа к данным.

## Технологический стек (Предполагаемый/Используемый)

*   **Веб-сервис:** Python (Django)
*   **База данных:** Clickhouse
*   **Контейнеризация:** Docker, Docker Compose

## Традиционный подход и его недостатки

В настоящее время для контроля деформаций и смещений оборудования применяется периодический мониторинг с привлечением маркшейдерских служб. Данная методика имеет ряд существенных ограничений:
*  Периодичность проверок: Замеры выполняются не чаще 1 раза в месяц
*  Трудоемкость процесса: Требуется участие минимум двух квалифицированных специалистов
*  Задержка в получении данных: Результаты становятся доступны только после обработки измерений
*  Высокие операционные затраты: Постоянные выезды специалистов на объект

## Преимущества автоматизированной системы
Разрабатываемая система обеспечивает переход от эпизодического контроля к непрерывному мониторингу в режиме реального времени:
*  Автоматизация измерений: Данные собираются с заданной периодичностью без участия персонала
*  Круглосуточный мониторинг: Система работает 24/7, обеспечивая постоянный контроль
*  Мгновенное оповещение: Уведомления о критических изменениях поступают немедленно
*  Исторический анализ: Накопление данных для прогнозной аналитики и trend-анализа
*  Снижение затрат: Минимизация необходимости регулярных выездов специалистов
